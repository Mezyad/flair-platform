Sprint 4 Retrospective

What went well during the sprint cycle?
  1. We fixed several home and about pages codes so the tests pass because we were focusing more on just coding without testing
  2. We worked on branches
  3. We managed to finish some tasks in less time than expected such as (deleting old events) and we used this time to meet the unexpected new tasks
What went wrong during the sprint cycle?
  1. We spent more time on making the tests pass than on coding new things
  2. We are having trouble and feeling desperate because some functions are working but still the test doesn’t pass and we don’t know why
  3. The new tasks added in the middle of our workload created a disruption to the plans we already had. As you can see in the graph the expectation is that we will not be able to finish the project on time (Pessimistic estimation)
What could we do differently to improve?
  1. From the beginning of our coding we should make sure that tests pass so we don’t lose extra time fixing code
  2. Create more pull requests even if they are small commits
  3. Agree on meeting at least once a week
  4. We should be better prepared to receive more unexpected tasks from the owners during our process
